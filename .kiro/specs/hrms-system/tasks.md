# План реализации: HRMS System

## Обзор

Реализация HRMS системы на базе Python и Excel с архитектурой "Excel как база данных, Python как процессор". Система запускается ТОЛЬКО из Excel через VBA макрос. Разработка ведется инкрементально: от базовой инфраструктуры к простым функциям, затем к сложным.

**Язык реализации**: Python 3.10+

**Ключевые принципы**:
- Маленькие проверяемые шаги - каждая задача тестируется сразу
- Переиспользуемые паттерны - создаем шаблоны для повторяющейся логики
- Инкрементальная разработка - каждый шаг работает и проверяется

## Важные технические замечания

### 1. Работа с датами Excel
Excel хранит даты как числа (дни с 1900 года), Python использует datetime объекты.
**Решение**: В `db_engine.py` конвертировать все даты сразу при чтении через pandas `pd.to_datetime()`.

### 2. Производительность xlwings
xlwings работает через COM, это медленно при чтении по ячейкам.
**Решение**: Читать весь диапазон за раз до последней заполненной строки (`sheet.used_range` или автоопределение), обрабатывать в памяти через pandas, записывать пакетом. Это работает для любого количества строк (100, 500, 1000+).

### 3. Запрещенные символы в именах папок
В ФИО могут быть символы, недопустимые в именах папок Windows.
**Решение**: Очищать строку от символов: `/ \ : * ? " < > |` перед созданием папки.

### 4. Приоритет критичных алертов
Красные алерты (контракты <= 7 дней) не должны теряться в списке.
**Решение**: Сортировка: сначала все HIGH по дате, затем MEDIUM по месяцам.

### 5. Виртуальное окружение
Изоляция зависимостей для переносимости проекта.
**Решение**: Использовать `.venv`, создать `setup.bat` для автоматической установки.

- Задачи с `*` являются опциональными (property-тесты) и могут быть пропущены для быстрого MVP
- Каждая задача ссылается на конкретные требования для трассируемости
- Checkpoints обеспечивают инкрементальную валидацию
- Property-тесты валидируют универсальные свойства корректности
- Unit-тесты валидируют конкретные примеры и граничные случаи
- Штатка (Фаза 13) ОТЛОЖЕНА и будет реализована в последнюю очередь

    data = self.db.get_employee(employee_id)

    doc = self.doc_gen.populate_template(type, data)
