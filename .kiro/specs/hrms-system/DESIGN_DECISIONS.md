# Ключевые решения по дизайну HRMS системы

## 1. Способ запуска
✅ **Решение**: Только из Excel через VBA макрос (кнопка в Excel → Python GUI)
❌ **Отклонено**: Отдельное .exe приложение

**Причина**: Не вижу смысла делать отдельное приложение, которое надо еще и открывать

## 2. Структура Excel файла

### Листы:
- **Сотрудники** - активные сотрудники (ручное добавление, Python редактирует/увольняет)
- **Архив** - уволенные сотрудники (Python управляет автоматически)
- **Отпуска** - журнал отпусков с привязкой к Таб. № (раньше в отдельном файле)
- **Журнал событий** - журнал всех сгенерированных приказов с автоинкрементом номеров
- **Настройки** - справочники (Должность, События, Подразделение, Форма оплаты, Ставка)
- **Штатка** - НЕ В ПРИОРИТЕТЕ, оставить на самый конец

### Лист "Журнал событий":
- Хранит все сгенерированные приказы
- Автоматическая нумерация (последний номер + 1)
- Возможность ручного ввода номера
- Колонки: Номер приказа, Тип события, Дата создания, Таб. №, ФИО, Путь к файлу

### Лист "Архив":
- Все колонки из "Сотрудники" + "Дата увольнения" + "Причина увольнения"
- Строка ПЕРЕМЕЩАЕТСЯ из "Сотрудники" в "Архив" при увольнении

## 3. Операции с сотрудниками

### ✅ Делаем автоматически:
- **Редактирование** - изменение любых полей существующего сотрудника (все поля можно менять)
- **Увольнение** - перемещение на лист "Архив" с датой увольнения
- **Продление контракта** - обновление даты "Конец контракта"

### ❌ НЕ делаем:
- **Добавление нового сотрудника** - только ручное добавление в Excel таблицу

## 4. Генерация приказов

### Текущий подход:
- Шаблоны Word уже есть у пользователя
- **Пока делаем ОДИН типовой приказ** с базовым набором данных
- Детали каждого типа приказа прорабатываем потом отдельно

### Нумерация приказов:
- **Автоматическая** с запоминанием последнего номера
- Хранение в листе "Журнал событий" (каждый приказ = строка)
- **Возможность ручного ввода** номера при создании приказа

### Типы приказов:
- Больничный
- Отпуск за свой счет
- Отпуск трудовой
- Перевод
- Прием на работу
- Продление контракта
- Увольнение

### Workflow:
- Все операции (увольнение, продление) идут через создание приказа
- Приказ генерируется → данные обновляются в Excel

## 5. Виджеты Dashboard

### Контракты истекают:
- **НЕ топ 5-10**, а **ВСЕ контракты на 2-3 месяца вперед**
- Группировка по месяцам
- С сортировками и фильтрами

### Дни рождения:
- **НЕ 2 недели**, а **месяц вперед** (30 дней)

### Статистика:
- Всего сотрудников
- По подразделениям (Завод КТМ, Основное)
- В отпуске

## 6. Подразделения

- **"Завод КТМ"** (не "Завод")
- **"Основное"** (не "Офис")

## 7. Названия колонок

- **БЕЗ подчеркиваний**, с обычными пробелами (как в Excel)
- Примеры: "Таб. №", "Дата рождения", "Форма оплаты"
- В pandas: `df['Дата рождения']` (не `df['Дата_рождения']`)

## 9. Личные дела сотрудников

### Хранение:
- **Путь указывается в настройках** (settings.py: `PERSONAL_FILES_DIR`)
- По умолчанию: `личные_дела/` (в той же папке что и Excel)
- Можно изменить на любой путь

### Структура папок:
- Формат: `{Таб. №} {ФИО}/` (БЕЗ подчеркиваний!)
- Пример: `001 Иванов Иван Иванович/`
- Путь хранится в колонке "Путь к личному делу" в листе "Сотрудники"

### Создание папки:
- **Автоматически** при генерации приказа о приеме на работу
- **Вручную** через кнопку "Создать личное дело" в карточке сотрудника
- Путь сохраняется в Excel и можно редактировать

### Функционал в GUI:
- **В карточке сотрудника**: Кнопка "Открыть личное дело" → открывает папку в проводнике
- **В списке сотрудников**: Иконка 📁 (если папка существует) → клик открывает папку
- Если папки нет: кнопка "Создать личное дело"

### Фаза 1 (критично):
1. Контроль контрактов - алерты на 2-3 месяца
2. Дни рождения - виджет на 30 дней
3. Управление отпусками - добавить/редактировать/удалить
4. Проверка пересечений отпусков

### Фаза 2 (важно):
5. Генерация приказов (один типовой)
6. Увольнение с перемещением в Архив
7. Продление контракта
8. Фильтрация по подразделениям

### Фаза 3 (потом):
9. Штатка с окладами
10. Детальная проработка каждого типа приказа
11. Расширенная аналитика
