# Дизайн: Журнал приказов

## 1. Overview

Добавление функционала журнала приказов в db_engine.py для автонумерации и логирования сгенерированных документов.

## 2. Структура данных

### Лист "Журнал событий"

| Колонка | Тип | Обязательно | Описание |
|---------|-----|-------------|----------|
| Номер приказа | String | Да | "001-П", "045-ОТ" |
| Тип события | String | Да | Из справочника "События" |
| Дата создания | Date | Да | Автозаполнение |
| ФИО | String | Да | Основная ссылка на сотрудника |
| Таб. № | Integer | Нет | Дополнительно |
| Путь к файлу | String | Нет | приказы/... |

### 3.2 get_next_order_number()

```python
def get_next_order_number(self, order_type: str) -> str:
    """
    Получение следующего номера для типа приказа
    
    Args:
        order_type: Тип события (например "Прием на работу")
        
    Returns:
        Следующий номер, например "001-П"
        
    Raises:
        ValueError: Если тип события не найден в справочнике
    """
```

**Логика:**
1. Читаем все записи журнала для данного типа
2. Находим максимальный номер
3. Извлекаем числовую часть регуляркой `(\d+)-[А-Яа-я]+`
4. Возвращаем "новый_номер-{код}"
5. Если журнал пустой → "001-{код}"

### 3.3 _get_type_code()

```python
def _get_type_code(self, order_type: str) -> str:
    """
    Получение кода типа из справочника
    
    Mapping из справочника "События":
    - Прием на работу → П
    - Увольнение → У
    - Отпуск трудовой → ОТ
    - Отпуск за свой счет → ОТБ
    - Больничный → Б
    - Перевод → ПЕР
    - Продление контракта → ПК
    
    Args:
        order_type: Название типа из справочника
        
    Returns:
        Код типа (2 символа max)
    """
```

### 3.4 add_order_log()

```python
def add_order_log(self, order_data: Dict) -> str:
    """
    Добавление записи в журнал приказов
    
    Args:
        order_data: {
            'order_type': str,      # Обязательно
            'search_value': str,    # Единое поле поиска (ФИО или Таб.№)
            'file_path': str        # Опционально
        }
        
    Returns:
        Номер созданного приказа
        
    Raises:
        ValueError: Если сотрудник не найден или тип события некорректен
    """
```

**Логика единого поиска сотрудника:**
```
Вход: search_value (строка)

1. Пытаемся преобразовать в int (для Таб.№)
2. Если число:
   - Ищем точное совпадение в "Таб. №"
   - Сохраняем как tab_number
3. Если строка (или число не найдено):
   - Ищем частичное совпадение в "ФИО" (case-insensitive)
   
Результат:
- 0 найдено → ValueError "Сотрудник не найден"
- 1 найдено → используем найденного
- >1 найдено → используем первого (или предупреждение)
```

**Процесс добавления:**
1. Валидируем order_type
2. Ищем сотрудника через единый поиск
3. Получаем следующий номер через get_next_order_number()
4. Записываем строку в журнал

## 4. Примеры использования

```python
db = ExcelDatabase()
db.connect()

# Получить следующий номер для приёма
next_num = db.get_next_order_number("Прием на работу")
# next_num = "001-П"

# Добавить запись в журнал
order_number = db.add_order_log({
    'order_type': 'Прием на работу',
    'search_value': 'Иванов',  # или '12345'
    'file_path': 'приказы/Прием Иванов Иван 2024-01-15.docx'
})
# order_number = "001-П"

# Получить все приказы
all_orders = db.get_order_log()

# Получить только приёмы
hires = db.get_order_log("Прием на работу")
```

## 5. Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Пустой журнал | Возвращает "001-{код}" |
| Тип не в справочнике | ValueError |
| Сотрудник не найден | ValueError "Сотрудник не найден: {search_value}" |
| Дубликат номера | Разрешаем (возможен ручной ввод) |
